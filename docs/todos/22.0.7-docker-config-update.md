# 📋 Tarefa 22.0.7 — Atualizar Docker e CI para nova estrutura

## Objetivo
Atualizar as configurações de Docker e Docker Compose para refletir a nova estrutura de pastas do backend e do frontend, garantindo que o build e a inicialização ocorram sem erros de caminhos de arquivos.

## Arquivos Afetados
- `backend/Dockerfile`
- `docker-compose.yml`

## Plano de Execução
1. Ler e analisar o `backend/Dockerfile` e `docker-compose.yml` existentes para identificar referências a caminhos de arquivos ou entry points antigos (por exemplo, `main:app` em vez de `src.main:app` ou montagem de volumes incorretas).
2. Atualizar o `backend/Dockerfile` para expor o novo entrypoint `src.main:app` ou similar e copiar a estrutura correta.
3. Atualizar `docker-compose.yml` ajustando volumes montados para o novo layout de pastas.
4. Validar o build e a inicialização local dos containers via Docker Compose.

## Verificação
- O build do Docker Compose é concluído com sucesso e todos os containers sobem sem erros.
