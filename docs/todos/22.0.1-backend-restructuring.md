# 📋 Tarefa 22.0.1 — Restruturação de Diretórios Backend

## Objetivo
Mover arquivos de código-fonte `.py` em `backend/` para subpastas estruturadas em `backend/src/` para separar as responsabilidades e limpar a raiz do diretório.

## Estrutura de Destino
- `backend/src/api/` — `router_*.py`
- `backend/src/core/` — `engine.py`, `evaluator.py`, `goalseek.py`
- `backend/src/db/` — `database.py`, `seeding.py`
- `backend/src/services/` — `services_*.py`, `exports.py`, `exports_harvest_plan.py` (essas serão renomeadas/unificadas depois, agora apenas moveremos os arquivos existentes)
- `backend/src/schemas/` — `schemas.py`
- `backend/src/` — `main.py`

## Ferramentas & Habilidades Relevantes
- `os-fundamentals`: Execução de comandos PowerShell seguros para movimentação de arquivos
- `systematic-debugging`: Para resolver erros de importação circulares ou relativos
- `python-patterns`: Para organizar imports corretos e verificar PEP8

## Plano de Execução
1. Criar subpastas em `backend/src/`.
2. Mover arquivos.
3. Atualizar todos os imports em todos os arquivos de backend (inclusive testes e scripts que referenciam o raiz do backend).
4. Rodar testes existentes via `pytest` para garantir que o comportamento é preservado.

## Verificação
- `pytest` deve passar.
